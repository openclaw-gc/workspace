### System Architecture

```
LAYER 1: SESSION CONTEXT
├─ Auto-load: SOUL.md, IDENTITY.md, USER.md
└─ Instant recall via memory_search()

LAYER 2: STRUCTURED MEMORY
├─ MEMORY.md (curated decisions)
├─ PROJECT-STATE.md (execution state)
├─ memory/YYYY-MM-DD.md (daily logs)
└─ memory/state.json (checkpoint state)

LAYER 3: AUDIT TRAIL (Immutable)
├─ memory/audit.jsonl (every decision logged)
├─ memory/skill-manifest.json (capabilities + versions)
└─ memory/checkpoint-state.json (current status)

LAYER 4: OPERATIONAL GUARDRAILS
├─ memory/guardrails.json (7 rules in JSON)
├─ guardrails-validator.js (pre-output validation)
└─ Enforcement: Manual validation on responses, automatic audit logging
```

### What This Solves

✅ **24/7/365 Coherence** — Cannot forget state because it's logged + checkpoint-enforced  
✅ **Execution Visibility** — You can see when working vs. idle via checkpoint status  
✅ **Operational Discipline** — Rules actively validated, not just documented  
✅ **Decision Audit Trail** — Every decision permanently logged with timestamp  
✅ **Capability Inventory** — Clear view of what's live vs. planned + roadmap
